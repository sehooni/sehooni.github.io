19:T88b,<p>자연어처리(NLP) 공부를 하면서 정리해놓은 <code>.ipynb</code>파일이 꽤나 많아졌다. 이러한 내용들을 블로그에 업로드하려면 결국 <code>.md</code>로 바꿔줘야 한다.</p>
<p>그렇다면 어떠한 방법을 사용해야할까?</p>
<h2><code>nbconvert</code></h2>
<p><code>nbconvert</code>는 jupyter notebook을 markdown 파일 혹은 html파일로 변환해주는 프로그램이다.</p>
<p>아래의 명령어를 통해 설치할 수 있다.</p>
<pre><code class="language-bash"># pip을 통해 nbconvert 설치
pip install nbconvert
</code></pre>
<pre><code class="language-bash"># conda를 통해 nbconvert 설치
conda install nbconvert
</code></pre>
<p>예를 들어 살펴보자.</p>
<p><code>Hello.ipynb</code>라는 jupyter notebook 파일을 markdown 파일로 변환하고 싶다면?
이를 위해 먼저 <code>Hello.ipynb</code> 파일이 있는 경로로 이동한다. 그 뒤, 아래의 명령어를 실행한다.</p>
<pre><code class="language-bash">jupyter nbconvert --to markdown Hello.ipynb
</code></pre>
<p>위의 명령어는 해당 경로에 다음의 두 가지 결과물을 생성한다.</p>
<ul>
<li><code>Hello.md</code> : markdown 파일</li>
<li><code>Hello_files</code> : markdown 파일에 포함되어 있는 모든 이미지들을 모아놓은 폴더</li>
</ul>
<p>이를 활용하여 Github 블로그에 포스팅 하기 위해서는 다음과 같은 작업을 거쳐야 한다.</p>
<ol>
<li><code>Hello.md</code> 파일 내 상단에 YAML 부분을 추가한다.</li>
<li><code>Hello.md</code> 파일 내 이미지들의 경로를 블로그의 이미지 경로 형태로 수정한다.</li>
<li><code>Hello.md</code> 파일의 파일명을 날짜(date)와 제목(title)을 포함하는 <code>YYYY-MM-DD-TITLE.md</code> 형태로 바꿔준다.</li>
<li>이 <code>YYYY-MM-DD-TITLE.md</code> 파일을 블로그의 <code>_posts</code> 폴더로 이동시킨다.</li>
<li><code>Hello_files</code> 폴더를 블로그의 <code>images</code> 폴더로 이동시킨다.</li>
</ol>
<p>이제 이 파일들을 모두 add/commit/push하면 Github 블로그에 포스팅이 추가되는 것을 확인할 수 있다.</p>
